Buttons

Buttons trigger actions or navigation. Use them for form submissions, dialog confirmations, or simple links styled as buttons.

<!-- Primary Button -->
<button type="button" class="btn btn--primary">Primary Button</button>

<!-- Secondary Button -->
<button type="button" class="btn btn--secondary">Secondary Button</button>

<!-- Disabled Button -->
<button type="button" class="btn btn--primary" disabled>Disabled Button</button>
/* Button Component */
.btn {
  display: inline-flex;
  align-items: center;
  justify-content: center;
  padding: var(--space-2) var(--space-4);
  border-radius: var(--radius-md);
  /* ... */
}
/* ... styles ... */

Available Modifiers:

  • .btn--primary: For the main action on a screen.
  • .btn--secondary: For alternative actions.
  • disabled attribute: To prevent interaction.

Accessibility:

  • Use <button> for actions.
  • Icon-only buttons require aria-label.

Form Inputs

Inputs allow users to enter data. They include built-in labels and support validation states.



<!-- Text Input -->
<div>
  <label for="username" class="form-label">Username</label>
  <input type="text" id="username" class="form-input" placeholder="Enter username">
</div>

<!-- Error State -->
<div>
  <label for="email" class="form-label">Email</label>
  <input type="email" id="email" class="form-input form-input--error" value="invalid">
</div>

<!-- Checkbox -->
<label for="subscribe" class="form-checkbox-label">
  <input type="checkbox" id="subscribe">
  Subscribe to newsletter
</label>
/* Input Component */
.form-label {
  display: block;
  font-size: var(--font-size-sm);
  margin-bottom: var(--space-2);
}

.form-input {
  display: block;
  width: 100%;
  padding: var(--space-2) var(--space-3);
  background-color: var(--color-surface);
  border: 1px solid var(--color-border);
  border-radius: var(--radius-md);
  transition: all var(--transition-base);
}

.form-input:focus {
  border-color: var(--color-primary);
  box-shadow: var(--shadow-focus);
}

.form-input--error {
  border-color: var(--color-error);
}

.form-checkbox-label {
  display: inline-flex;
  align-items: center;
  gap: var(--space-2);
  cursor: pointer;
}

Available Modifiers:

  • .form-input--error: Highlights the border in red to indicate validation errors.
  • .form-checkbox-label: A wrapper to correctly align checkboxes with their labels.

Accessibility:

  • Always pair <label> with <input> using for/id.
  • Add aria-invalid="true" for error states.
  • Use aria-describedby to link error text to inputs.

Structure:

  • .card: The main container.
  • .card__title: The heading area.
  • .card__body: The main content area (stretches to fill space).
  • .card__footer: The bottom area for actions.

Accessibility:

  • Use semantic tags like <article>.
  • Ensure logical heading hierarchy within cards.

Badges

Badges are compact elements used to display metadata, status, or tags. They are often found inside tables or cards.

Default Primary Success Warning Error 
<span class="badge">Default</span>
<span class="badge badge--primary">Primary</span>
<span class="badge badge--success">Success</span>
<span class="badge badge--warning">Warning</span>
<span class="badge badge--error">Error</span>
/* Badge Component */
.badge {
  display: inline-flex;
  align-items: center;
  padding: var(--space-1) var(--space-2);
  font-size: var(--font-size-xs);
  font-weight: var(--font-weight-medium);
  border-radius: var(--radius-full);
  background-color: var(--color-surface-hover);
  color: var(--color-text-main);
}

.badge--primary {
  background-color: var(--color-primary);
  color: var(--color-text-inverse);
}

.badge--success { background-color: var(--color-success); }
.badge--warning { background-color: var(--color-warning); }
.badge--error { background-color: var(--color-error); }

Available Modifiers:

  • .badge--primary: Standard brand color.
  • .badge--success: Green for positive status.
  • .badge--warning: Yellow for caution.
  • .badge--error: Red for critical status.

Accessibility:

  • Ensure sufficient text color contrast.
  • If color alone conveys meaning, ensure text context is sufficient.

Alerts

Alerts communicate important messages to the user. They use distinct left-border colors to indicate priority.

Info: This is an informational alert.
Success: Operation completed successfully.
Warning: Please check your input.
Error: Something went wrong. 
<!-- Info Alert -->
<div class="alert alert--info" role="alert">
  <strong>Info:</strong> This is an informational alert.
</div>

<!-- Success Alert -->
<div class="alert alert--success" role="alert">
  <strong>Success:</strong> Operation completed.
</div>

<!-- Error Alert -->
<div class="alert alert--error" role="alert">
  <strong>Error:</strong> Something went wrong.
</div>
/* Alert Component */
.alert {
  padding: var(--space-4);
  border-radius: var(--radius-md);
  border: 1px solid var(--color-border);
  border-left-width: 4px;
  background-color: var(--color-surface);
  color: var(--color-text-main);
  margin-bottom: var(--space-4);
}

.alert--info { border-left-color: var(--color-primary); }
.alert--success { border-left-color: var(--color-success); }
.alert--warning { border-left-color: var(--color-warning); }
.alert--error { border-left-color: var(--color-error); }

Available Modifiers:

  • .alert--info: Blue for general information.
  • .alert--success: Green for success messages.
  • .alert--warning: Yellow for warnings.
  • .alert--error: Red for errors or critical failures.

Accessibility:

  • Use role="alert" (assertive) for immediate screen reader announcements.
  • Use role="status" (polite) for passive updates.

Modal

Modals are dialog overlays that demand user attention. They disable interaction with the rest of the page until closed. This implementation uses CSS-only :target selectors.

Open Demo Modal

Confirm Action

×

Are you sure you want to proceed with this action? This can be undone if you try hard enough.

Cancel Confirm 
<!-- Trigger Button -->
<a href="#demo-modal" class="btn btn--primary">Open Demo Modal</a>

<!-- Modal Overlay & Content -->
<div id="demo-modal" class="modal-overlay modal">
  <div class="modal__content">
    <div class="modal__header">
      <h3 class="modal__title">Confirm Action</h3>
      <a href="#" class="modal__close">&times;</a>
    </div>
    <div class="modal__body">
      <p>Msg...</p>
    </div>
    <div class="modal__footer">
      <a href="#" class="btn btn--secondary">Cancel</a>
      <a href="#" class="btn btn--primary">Confirm</a>
    </div>
  </div>
</div>
/* Modal Component (CSS-Only) */
.modal-overlay {
  position: fixed;
  inset: 0;
  background-color: var(--color-overlay);
  display: flex;
  align-items: center;
  justify-content: center;
  z-index: 1000;
  opacity: 0;
  pointer-events: none;
  transition: opacity var(--transition-base);
}

.modal:target {
  opacity: 1;
  pointer-events: auto;
}

.modal__content {
  background-color: var(--color-surface);
  border-radius: var(--radius-lg);
  padding: var(--space-8);
  width: 90%;
  max-width: 500px;
}

Structure:

  • .modal-overlay, .modal: The full-screen background overlay.
  • .modal__content: The centered dialog box.
  • .modal__close: Standard close "X" button.

Accessibility Notes (CSS-Only Limit):

  • This demo lacks focus trapping and "Esc" key support (requires JS).
  • For production: Add role="dialog" and aria-modal="true".
  • Ensure focus moves *into* the modal when opened and *back* to trigger when closed.
